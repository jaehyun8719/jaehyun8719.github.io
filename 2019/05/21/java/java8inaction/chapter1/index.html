Java8 In Action - 1. 동작 파라미터화 코드 전달하기

Posted on 2019-05-21 In Java , Java8 In Action Disqus:

Java8 In Action 을 읽고 정리한 내용이다. 동작 파라미터화(behavior parameterization)를 이용하면 자주 바뀌는 요구 사항에 효과적으로 대응할 수 있다. 이번 장에서는 기존에 사용했던 방식과 람다로 사용했을 시 변화되는 코드를 확인한다.

변화하는 요구사항에 대응하기

농장 재고 목록에 애플리케이션 리스트에서 녹색(green) 사과만 필터링하는 기능 추가

녹색 사과 필터링

단순히 녹색(green) 사과만 필터링하는 소스이다.

실행 결과

여기서 빨간 사과도 필터링하고 싶어질 때, 생각을 하지 않으면 단순히 복사 후 if ("green".equals(apple.getColor())) 이 부분에서 "green" 을 "red"로 변경해주면 된다. 하지만 여기서 다른 여러 가지 색깔을 필터링을 하려면 이와 같이 반복작업을 진행해야 된다. 그래서 비슷한 코드를 구현한 다음에 추상화를 해야 된다.

동작 파라미터화

요구사항에 좀 더 유연하게 대응할 수 있는 방법이 필요했다. 선택 조건을 결정하는 인터페이스를 정의 해보자.

아래와 코드와 같이 여러 조건을 정의할 수 있다.

  • 전략 디자인 패턴을 이용하면 ApplePredicate가 알고리즘 패밀리고 AppleHeavyWeightPredicate와 AppleGreenColorPredicate가 전략이다.

추상적 조건으로 필터링

  • 추가 요구 사항 시 ApplePredicate를 정의해서 test 메서드를 수정만 해주면 된다.

복잡한 과정 간소화

filterApples 메서드로 새로운 동작을 전달하려면 ApplePredicate 인터페이스를 구현하는 여러 클래스를 정의한 다음에 인스턴스화해야 한다는 단점이 있다. 여기서 단점을 보안하기 위해서는 익명클래스(anonymous class)를 이용하면 된다.
